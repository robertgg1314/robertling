Anthropic MCP 的架构包含哪些部分

Anthropic 的模型上下文协议（MCP）架构主要包含以下几个部分：MCP 主机，包括 Claude Desktop、IDE 等需要通过 MCP 访问资源的 AI 工具；MCP 客户端，与服务器保持一对一连接的协议客户端；MCP 服务器，一个轻量级程序，通过标准化的 MCP 协议开放特定功能；本地资源，如计算机上的数据库、文件和服务等资源，MCP 服务器可以安全地访问这些内容；远程资源，通过互联网访问的 API 等资源，MCP 服务器可以与之建立连接。

MCP 的架构设计旨在打破信息孤岛，提升 AI 系统与业务工具和数据源之间的双向连接能力。通过这样的架构，AI 工具能够更加高效地获取所需的信息资源，从而提高响应的质量和相关性。例如，在开发环境中，开发者可以利用 Claude Desktop 等 MCP 主机，通过 MCP 客户端与服务器连接，从而访问本地数据库或远程的 API 资源，实现快速的代码开发和项目管理。这种架构不仅为开发者提供了便利，也为企业在信息管理和工作效率方面带来了重大的变革。

Energy-based SSL（基于能量的半监督学习）是一种在半监督学习领域的方法。

其核心概念是利用能量分数来衡量未标记样本是否可能“分布内”，即接近于当前的训练数据。它不依赖于模型置信度，而是通过这种方式来确定未标记样本是“分布中”还是“分布外”。

例如，在不平衡半监督学习中，采用分布内伪标记优先的方法（如InPL），利用能量得分来判断未标记样本的情况。通过计算每个未标记样本的能量分数，只有当能量低于某个阈值时，才用模型生成的预测类对其进行伪标记。

这种方法的优势包括简便、有效且使用广泛。简便在于和模型置信度的指标具有相似的简洁性；有效是因为能量得分与数据样本的概率密度一致，能较好地反映样本是否符合训练集的分布；使用广泛体现在以往的OOD目标检测工作中得到了较多应用。

重要亮点

  • 随机变量的定义：随机变量是一个将样本空间中的结果映射到实数的函数。比如在掷骰子的例子中，每个面被映射到一个数字。在抛两枚硬币的例子中，可定义一个随机变量来计算正面的数量。
  • 随机变量的作用：它简化了对复杂结果的处理，是连接现实世界随机性和数学秩序的接口，是一种抽象层，能让我们专注于关心的数值属性，剥离不相关细节，聚焦于数值上重要的内容。
  • 与样本空间的关系：样本空间是实验所有可能结果的集合，在处理复杂问题时，如两个骰子的点数和，不使用随机变量就得不断参考原始结果，会很混乱，而随机变量能简化问题。
