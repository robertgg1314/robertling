  • (神经网络语言模型) NNLM 使用神经网络预测词序列中下一个词的概率，是早期词嵌入的雏形。
  • (Word2Vec) Word2Vec 使用 CBOW 和 Skip-gram 模型，将词表示为稠密向量，捕捉词语的语义关系。
  • (GloVe) GloVe 基于全局词共现统计，结合 Word2Vec 优点，学习词向量。
  • (FastText) FastText 将词表示为子词序列的组合，学习词的形态信息，处理未登录词。
  • (ELMo) ELMo 使用双向 LSTM 学习上下文相关的词向量，生成不同的向量表示。
  • (BERT) BERT 基于 Transformer 架构，捕捉长距离上下文依赖，标志预训练模型时代。
  • (改进的预训练模型) RoBERTa、ALBERT、XLNet 等在模型架构和训练方法上优化，提升 NLP 性能。

FastText 的核心思想是将一个单词表示为其组成字符 n-gram 的集合，然后将这些 n-gram 的向量表示求和或平均，作为该单词的最终向量表示。
举例说明：
假设单词是 "where"，n-gram 的范围是 3 到 6，则该单词的 n-gram 集合为：
• 3-gram: "<wh", "whe", "her", "ere", "re>"
• 4-gram: "<whe", "wher", "here", "ere>"
• 5-gram: "<wher", "where", "here>"
• 6-gram: "<where", "where>"
其中，"<" 和 ">" 是特殊的边界符号，用于标记单词的开头和结尾。
FastText 会为每个 n-gram 学习一个向量表示。然后，将单词 "where" 的所有 n-gram 的向量表示求和或平均，得到 "where" 的最终向量表示。

Cursor：如何构建 AI Coding 最佳实践？

构建 AI Coding 最佳实践的方法包括以下方面：

  1. 开发专门的小模型：围绕代码预测、补齐等各类任务训练专门的小模型，以提高特定任务的性能。
  2. 采用推测编辑：从推测解码衍生出推测编辑，通过并行处理提高模型速度，减少延迟。
  3. 优化用户体验：如设计独特的 Tab 键逻辑，让模型自动预测并跳转到下一个可能需要编辑的位置，实现更流畅的编辑体验；还包括优化 diff 视图，根据不同任务和场景提供合适的界面。
  4. 结合多种模型：除了使用最新一代模型，Cursor 还训练了自定义模型，如在 Apply 任务中，利用自定义模型勾勒代码块并应用更改，以更好地显示 diff。
  5. 提高响应速度：采用缓存预热策略，预判用户可能用到的上下文信息并预热缓存；在 Cursor Tab 设计中预测用户可能接受的建议，提前处理下一个请求操作；使用高效的注意力机制，压缩 keys 和 values 以提升速度，增加缓存容量和命中率，支持更长的 prompt。
  6. 以网页设计为灵感的 Priompt 工程：借鉴现代网页开发实践，Priompt 系统可以合理排布内容，应对有限的上下文窗口问题，将数据处理和展示分离，方便 debug，还可根据代码行的重要性设置优先级。
  7. 考虑模型的优势与不足：如 Claude 3.5 Sonnet 综合实力较强，能理解开发者模糊需求并给出适当建议；即便是 SOTA 模型也不擅长找 bug，这是产品的机会。
  8. 重视 code base indexing：团队投入大量精力进行 code base indexing，为其他代码任务提供基础。
  9. 探索新的产品概念：如试验 Shadow Space 概念，在后台运行隐藏窗口让 AI 进行 coding 任务，不影响开发者操作。
  10. 创新工作流设计：例如设计智能的工作流，让模型进行规划，再由其他模型处理具体任务。
  11. 关注用户体验和反馈：未来编程中自然语言和代码将共存，根据具体任务选择最有效交互；通过真实用户的使用反馈评估模型性能，而非仅依赖基准测试。
  12. 解决 models routing 问题：目前底座模型和自有模型间可初步实现模型切换，但不同模型（如 GPT-4o、Claude sonnet 和 o1）间的切换较复杂，需进一步探索解决方案。
